# NRDB 사용 사례
## NRDB를 사용하면 좋은 경우
- 대량의 데이터를 사용하는 경우
- 낮은 대기시간, 응답시간을 요구하는 경우
	- 예를 들어 온라인 게임, 쇼핑
## NRDB를 사용하면 안 좋은 경우
- 데이터의 중복을 허용하면 안되는 경우
	- 예를 들어 금융, 회계
